# スクリプト構造ドキュメント

このディレクトリには、`Assets/Scripts/Runtime` 配下の機能ごとの構造ドキュメントを格納しています。

## 概要

プロジェクトはクリーンアーキテクチャに基づいたレイヤー構造を採用しており、各機能が複数のレイヤーに分散して実装されています。

### フォルダ構成

- **[InGame.md](./InGame.md)**: バトル、音楽同期、キャラクター、カメラなどのインゲーム主要ロジック。
- **[Persistent.md](./Persistent.md)**: 入力バッファリング、シーン遷移、共通BGM再生など、シーンを跨いで存在する機能。
- **[Player.md](./Player.md)**: スキル、プレイヤー進行データなどの永続的なプレイヤー関連機能。
- **[Utility.md](./Utility.md)**: 優先度付きキュー、リングバッファ、定数などの汎用機能。

## レイヤーの定義

| レイヤー名 | 役割 |
| :--- | :--- |
| **0. Utility** | 汎用クラス、データ構造、プロジェクト全域で使われる定数。 |
| **1. Domain** | 純粋なビジネスロジック、エンティティ、仕様。Unityに依存しない。 |
| **2. Application** | ユースケース、ドメインのオーケストレーション。具体的なゲームルール。 |
| **3. Adaptor** | インターフェースの変換、コントローラー、プレゼンター。 |
| **4. View** | UnityのGameObject、UI、エフェクト、アニメーションの制御。 |
| **5. InfraStructure** | 外部実装（ファイル保存、ネットワーク、Unity特定機能の実装）。 |
| **6. Composition** | 依存関係の注入、オブジェクトの初期化、エントリポイント。 |
